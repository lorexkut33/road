## Docker
**Docker** -  это инструмент, который контейнизирует проект со всеми зависимостями(прописываются в Dockerfile)
**Контейнер** -  это запущенный экземпляр образа. Это изолированный процесс в ОС
**image/образ** -  это неизменяемый слепок приложения со всеми зависимостями
**Dockerfile** -  это инструкция по сборке образа
**DockerHub** -  это репозиторий, где хранятся готовые образы(Nginx, python и тд)

## virtual machine/container
**Виртуальные машины**
ВМ эмулируют целое «железо». На одну физическую машину устанавливается гипервизор (Type 1 или Type 2), который создает несколько виртуальных компьютеров. У каждой ВМ есть своя полноценная гостевая ОС, свое ядро, драйверы и выделенные ресурсы (RAM, CPU).

**Контейнеры**
Контейнеры не запускают собственную ОС. Они используют ядро хостовой операционной системы и изолируют только процессы приложения. Для этого используются технологии ядра Linux: namespaces (для изоляции имен, сетей, пользователей) и cgroups (для ограничения ресурсов).

А вообще для идеальной защиты, контейнеры запускают на виртуальной машине

## Базовый рабочий процесс (Workflow)
Процесс работы с Docker обычно выглядит так:

1. Пишем Dockerfile: описываем среду (например: возьми Python, скопируй код, установи зависимости из requirements.txt).

2. Сборка (build): превращаем Dockerfile в Image.

3. Хранение (push): отправляем образ в реестр (Docker Hub или локальный Registry).

4. Запуск (run): скачиваем образ на сервер и запускаем контейнер.

## вот список команд, которые должны отскакивать от зубов:

docker build -t my-app . — собрать образ из текущей папки.

docker run -p 8080:80 my-app — запустить контейнер, пробросив порт 8080 хоста на порт 80 контейнера.

docker ps — список запущенных контейнеров.

docker images — список скачанных образов.

docker exec -it <container_id> bash — «войти» внутрь запущенного контейнера.

docker rm -f $(docker ps -aq) — остановить и удалить все контейнеры (полезно при очистке).
